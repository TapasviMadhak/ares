🔵 OPTION 2: Setup AI Model (One-time, ~10 minutes)
────────────────────────────────────────────────────────────
When you want AI-powered analysis:

    cd /mnt/d_drive/ares
    ./setup_ollama.sh

What it does:
  • Downloads llama3.2:3b model (~2GB)
  • Stores in /mnt/d_drive (not your full / partition)
  • Takes 5-10 minutes
  • Only run once
  • Enables AI features in future scans

After this, ./START.sh will automatically use AI.

📁 SCRIPT DESCRIPTIONS:
═══════════════════════════════════════════════════════════

setup_ollama.sh
  • Run ONCE when you want AI features
  • Downloads model to /mnt/d_drive
  • Not required for basic scanning

START.sh
  • Starts Ollama (with AI model if available)
  • Starts Backend API on port 8000
  • No frontend dashboard
  • Use this every time you want to start ARES

STOP.sh
  • Stops Backend
  • Stops Ollama
  • Cleans up all processes

test_ares.sh
  • Runs comprehensive tests
  • Tests health, API, and scanning
