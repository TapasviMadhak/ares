WORKFLOW EXECUTION:
-------------------

1. Initialization
   └─> Create Orchestrator(scan_id, target)
   └─> Initialize Redis connection
   └─> Load scan context

2. Workflow Execution
   └─> For each phase (recon, enum, vuln, exploit):
       ├─> ToolSelector.select_tools(context, phase)
       │   ├─> Get available tools from Hexstrike
       │   ├─> Filter by phase categories
       │   ├─> AI analysis via Ollama
       │   └─> Return prioritized tool list
       │
       ├─> Execute selected tools
       │   ├─> Check Redis cache
       │   ├─> Build parameters (AI-assisted)
       │   ├─> Execute via HexstrikeClient
       │   ├─> Cache results in Redis
       │   └─> Log to database
       │
       ├─> Process results
       │   ├─> Extract vulnerabilities
       │   ├─> Update scan context
       │   ├─> Save to database
       │   └─> Update effectiveness metrics
       │
       └─> Adapt strategy
           ├─> Analyze findings
           ├─> Adjust next phase tools
           └─> Log decision

3. Completion
   └─> Generate summary
   └─> Update scan status
   └─> Cleanup resources
